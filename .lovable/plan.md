

## Problem

The Support page in the live preview still shows **"Small Tip — Buy us a coffee ☕"** instead of **"Support Development"**. 

**Root cause**: There is a stale pre-built bundle at `public/assets/Support-CSikK-Om.js` (and related translation bundles like `useTranslation-2Adup3ba.js`) that contains the old text via an i18n translation layer. The preview appears to serve from these pre-built assets rather than building fresh from the `src/pages/Support.tsx` source, which already has the correct text.

## Plan

1. **Delete stale `public/assets/` bundles** — Remove all files in `public/assets/` that are pre-built production bundles from a previous build system. These are overriding the live Vite dev server output and causing the source changes to be invisible. Key files include `Support-CSikK-Om.js`, `useTranslation-2Adup3ba.js`, and all other `.js`/`.css` chunks in that directory.

2. **Verify the Support page renders from source** — After removing the stale assets, confirm the `/support` page now shows "Support Development" with the description "A small gesture that keeps the lights on." as written in `src/pages/Support.tsx`.

## Technical Details

- The `public/assets/` directory contains ~50+ pre-built JS/CSS chunks from a prior production build. These are not generated by the current Vite dev server and should not exist in the `public/` folder (Vite serves its own builds from memory during dev).
- The translation layer (`useTranslation`) referenced in the old bundle doesn't exist in the current source code — it's a relic of the old build.
- This fix will also resolve the "Guest Mode" card issue mentioned earlier, as that text likely also lives only in these stale bundles.
